Free-Fermion Volume Benchmark
=============================

This page summarizes the Gaussian (free-fermionic) volumetric benchmark
implemented in ``frontier.freefermionvolumebenchmark``.

.. contents::
   :local:
   :depth: 2

Background
----------

The FFV benchmark exercises circuits generated by quadratic fermionic
Hamiltonians, which act as orthogonal transformations on Majorana modes and can
be simulated efficiently on classical hardware. The benchmark preserves the
desirable properties of the Clifford volume benchmark—classical verifiability,
platform independence, and relevance to near-term algorithms—while probing a
distinct family of matchgate circuits drawn from $\mathrm{SO}(2n)$.

Benchmark Task
--------------

Free-Fermion Systems and Majorana Modes
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

A system of $n$ fermionic modes has annihilation/creation operators obeying the
canonical anti-commutation relations. By defining $2n$ Majorana operators,
quadratic Hamiltonians correspond to rotations in $\mathrm{SO}(2n)$, parameterized
by Givens rotations plus a diagonal $\pm 1$ phase matrix.

Random Free-Fermion Instances
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

Each sample draws a random orthogonal matrix $O \in \mathrm{SO}(2n)$ (using
``scipy.special_ortho_group``), decomposes it into Givens rotations and a diagonal
matrix $D$, and synthesizes the corresponding Gaussian circuit. A random
Majorana index is chosen to set the input state.

Benchmark Protocol
------------------

1. **Draw** a random orthogonal matrix $O$ and a random state index.
2. **Decompose** $O$ into Givens rotations (generate the Gaussian unitary) and a
   diagonal Pauli correction $D$.
3. **Build** the circuit: prepare the state indicated by the index, apply the
   Givens rotation gadgets, then apply the Pauli correction.
4. **Measurement circuits:** identify the Majorana modes with the largest weight
   for the chosen column of $O$, and generate measurement circuits for those
   observables.
5. **Execution:** run the circuits, collect counts, and attach them via
   :meth:`Benchmark.add_experimental_results`.

Performance Criteria
--------------------

The evaluation aggregates the expectation values into two categories:

* **Parallel values:** observables aligned with the chosen column of $O$.
* **Orthogonal values:** complementary observables.

Passing requires both sets of expectation values to meet thresholds derived
from the ideal parallel/orthogonal decomposition.

Implementation Notes
--------------------

Circuit Synthesis
^^^^^^^^^^^^^^^^^

Each Givens rotation is mapped to a native gate gadget (single- or two-qubit)
that acts on the encoded qubits. Phase corrections from $D$ become single-qubit
Pauli gates.

Measurement of Majorana Operators
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

Measurement circuits rephrase the selected Majorana operator as a Pauli string,
apply the inverse of the preparation circuit, and measure in the computational
basis so that bitstrings can be converted into $\pm 1$ expectation values.

Benchmark Score and Interpretation
----------------------------------

:meth:`FreeFermionVolumeBenchmark.evaluate_benchmark` returns per-sample
parallel/orthogonal expectation values, stores per-circuit expectation values
and standard errors, and provides plotting helpers for the distributions.

Usage
-----

1. Instantiate :class:`frontier.FreeFermionVolumeBenchmark` with the desired
   width, sample size, and shot count.
2. Call :meth:`create_benchmark` to generate the Gaussian circuits.
3. Execute the measurement circuits and attach the results.
4. Evaluate using :meth:`evaluate_benchmark` and inspect the `parallel_values`
   and `orthogonal_values` fields for scoring.
